# Catálogo Maestro de Competencias
## Subsistema 2 del Modelo Integral — Sistema de Competencias

## Alcance y decisión metodológica

El diseño original estimaba entre **175 y 230 competencias** (una por cada tarea o
subtarea elemental de cada curso). Solo se llegaron a codificar 6 competencias de ejemplo
antes de que el proyecto se desviara — y, al revisarlas, ni siquiera eran consistentes
entre sí: cada archivo del Lote 1 usó "COMP-001" para ilustrar un ejemplo distinto.

**Decisión para esta versión:** en vez de perseguir la cifra original de 175-230 (que
repetiría el mismo patrón de sobre-alcance ya identificado varias veces en este proyecto),
se define **una o dos competencias nucleares por curso** — 27 en total —, que representan
el resultado principal de aprendizaje de cada uno. Esto es suficiente para que el campo
"Competencia derivada" del IARF y de otros documentos deje de decir "Pendiente", y ofrece
una base sólida y ya usable. La desagregación en sub-competencias más finas (hasta llegar,
eventualmente, a un número mayor) queda como fase futura, solo si se necesita para fines
específicos (por ejemplo, certificación individual detallada ante el EC0060).
