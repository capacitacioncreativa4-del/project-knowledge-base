# ADR-0001: Selección de Arquitectura Docs-as-Code y Estrategia AI-First

## Contexto
La institución requiere un sistema centralizado para gestionar el conocimiento técnico de múltiples proyectos simultáneos[cite: 1]. Los enfoques tradicionales basados en wikis web o carpetas compartidas de documentos binarios sufren de degradación de contenido, falta de control de versiones fino y son difíciles de procesar de forma automatizada por herramientas de IA[cite: 1].

## Decisión
Hemos decidido adoptar una estrategia de **Documentación como Código (Docs-as-Code)** utilizando Markdown como estándar de formato, Git como sistema de control de versiones y GitHub como plataforma central de gobierno[cite: 1]. Además, cada documento integrará metadatos YAML estructurados en su parte superior para facilitar la interoperabilidad con agentes de IA autónomos[cite: 1].

## Consecuencias
* **Positivas**: Trazabilidad completa mediante commits informales/formales, control total de versiones de los activos y preparación nativa para automatizaciones en el futuro con herramientas como Codex[cite: 1].
* **Negativas**: Curva de aprendizaje inicial para colaboradores no familiarizados con Git o la consola de comandos[cite: 1].
