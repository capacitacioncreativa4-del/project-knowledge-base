# Principios Fundamentales del PKB

Este documento detalla las directrices inquebrantables que rigen la creación, modificación y auditoría de cualquier activo informacional dentro del ecosistema.

## 1. El conocimiento como activo institucional
Los documentos representan el valor intelectual de la organización. El contenido útil y estructurado tiene estricta prioridad sobre cualquier adorno o formato complejo[cite: 1].

## 2. Única fuente de verdad (SSOT)
Cada dato, política, estándar o decisión técnica debe existir en un único lugar del repositorio[cite: 1]. Se prohíbe explícitamente la duplicidad para evitar desincronizaciones[cite: 1].

## 3. Filosofía Docs-as-Code
La documentación se gestiona formalmente bajo el mismo estándar operativo que el software moderno: versionamiento con Git, aislamiento en ramas, control de cambios, Pull Requests y liberaciones estables (Releases)[cite: 1].

## 4. Diseño AI-First
Toda la documentación se construye estructurada semánticamente para asegurar la máxima compatibilidad de lectura e indexación automatizada por parte de agentes inteligentes y modelos de lenguaje (LLMs)[cite: 1].
