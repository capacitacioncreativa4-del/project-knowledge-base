# Extraction Guide — Framework v1

Este documento define el procedimiento oficial de extracción para asegurar que ninguna decisión valiosa se diluya.

## Procedimiento Operativo

### Paso 1: Identificar la conversación
Seleccionar el canal de chat o transcripción histórica dentro del inventario maestro.

### Paso 2: Segmentar en Sesiones
Dividir la conversación física en bloques lógicos o "sesiones de ingeniería" enfocadas en un solo tema o hito.

### Paso 3: Análisis de Capas
Para cada sesión de ingeniería, extraer e identificar rigurosamente:
* Objetivo
* Decisiones
* Entregables / Artefactos
* Pendientes
* Relaciones

### Paso 4: Construir la Ficha de Extracción
Poblar la plantilla oficial en la carpeta `sessions/` asignando el ID correspondiente[cite: 4].

### Paso 5: Revisión de Calidad
Aplicar la lista de verificación institucional (`REVIEW-CHECKLIST.md`)[cite: 4].

### Paso 6: Generar Objetos PKB
Tras la luz verde, persistir los objetos en el registro maestro e instanciar sus archivos Markdown individuales[cite: 4].
