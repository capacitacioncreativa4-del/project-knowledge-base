# Inventario Analítico de Requisitos Formativos (IARF)

## PARTE II

# Sistema de Descomposición Normativa (SDN)

---

## 11.1 Justificación

Durante muchos años los programas de capacitación para seguridad privada han presentado una limitación importante: toman un artículo de la ley y lo convierten directamente en un tema de un curso.

Por ejemplo:

> **Artículo X**
>
> "El personal deberá conducirse con profesionalismo."

Y el curso simplemente incorpora un tema denominado:

> Profesionalismo.

Desde una perspectiva de ingeniería curricular, este procedimiento es insuficiente.

Un artículo legal rara vez contiene un único aprendizaje. Normalmente integra diversos principios, obligaciones, límites y criterios de actuación.

Nuestro sistema trabajará con una **unidad mínima de análisis**, denominada **Elemento Normativo Formativo (ENF)**.

---

# 11.2 Elemento Normativo Formativo (ENF)

Un ENF es la expresión mínima de una obligación normativa que puede transformarse en un resultado de aprendizaje observable y evaluable.

Cada ENF deberá cumplir cuatro condiciones:

* Derivar de una fuente normativa identificada.
* Tener impacto directo o indirecto sobre el desempeño.
* Poder convertirse en una competencia.
* Ser susceptible de evaluación mediante evidencias.

---

## Ejemplo conceptual

Supongamos una disposición que establezca:

> "El personal actuará con legalidad, honradez, profesionalismo, eficiencia y respeto a los derechos humanos."

Aunque se presente como un solo enunciado, el SDN identifica múltiples ENF:

| ENF                           | Competencia preliminar                                                 |
| ----------------------------- | ---------------------------------------------------------------------- |
| Actuar con legalidad          | Aplicar el marco jurídico en el servicio.                              |
| Actuar con honradez           | Preservar la integridad y la confianza institucional.                  |
| Actuar con profesionalismo    | Ejecutar procedimientos conforme a estándares establecidos.            |
| Actuar con eficiencia         | Optimizar recursos y tiempos sin comprometer la seguridad.             |
| Respetar los derechos humanos | Proteger la dignidad e integridad de las personas durante el servicio. |

Cada ENF se convertirá posteriormente en criterios de desempeño, actividades didácticas y evidencias.

---

# 11.3 Taxonomía de ENF

Propongo clasificar los elementos normativos en diez categorías.

| Código | Categoría                | Finalidad                                           |
| ------ | ------------------------ | --------------------------------------------------- |
| ENF-J  | Jurídico                 | Define obligaciones y límites legales.              |
| ENF-OP | Operativo                | Regula procedimientos del servicio.                 |
| ENF-ET | Ético                    | Establece principios de conducta.                   |
| ENF-DH | Derechos humanos         | Garantiza el respeto a la persona.                  |
| ENF-CM | Comunicación             | Regula el intercambio de información.               |
| ENF-GD | Gestión documental       | Establece requisitos de registros e informes.       |
| ENF-SE | Seguridad y emergencias  | Define la respuesta ante incidentes.                |
| ENF-AT | Atención al usuario      | Regula la interacción con clientes y visitantes.    |
| ENF-TE | Tecnología y equipos     | Determina el uso adecuado de recursos tecnológicos. |
| ENF-GC | Gestión del conocimiento | Regula la actualización y mejora continua.          |

Esta taxonomía facilitará el análisis sistemático de las cinco fuentes normativas.

# 11.4 Ficha del Elemento Normativo Formativo (FENF)

Cada ENF se documentará mediante una ficha normalizada.

### Identificación

* Código ENF.
* Fuente normativa.
* Artículo.
* Fracción.
* Inciso.
* Tema.

### Naturaleza

* Obligación.
* Prohibición.
* Principio.
* Procedimiento.
* Requisito.
* Facultad.

### Competencia asociada

Código y descripción.

### Proceso operativo relacionado

Referencia al Catálogo Maestro de Procesos Operativos.

### Riesgo mitigado

Descripción y clasificación.

### Nivel de criticidad

Escala de 1 a 5.

### Evidencias

* Conocimiento.
* Desempeño.
* Producto.
* Resultado.

### Instrumentos de evaluación

* Examen.
* Caso.
* Lista de cotejo.
* Guía de observación.
* Simulación.
* Rúbrica.

# 11.5 Descomposición Multinivel

Cada disposición normativa será analizada en cinco niveles sucesivos.

```text
Nivel 1
Disposición normativa

        │
        ▼

Nivel 2
Elementos Normativos Formativos (ENF)

        │
        ▼

Nivel 3
Competencias

        │
        ▼

Nivel 4
Resultados de aprendizaje

        │
        ▼

Nivel 5
Criterios de desempeño
```

Con este procedimiento evitaremos que una disposición compleja quede reducida a un único contenido temático.

---

# 11.6 Índice de Densidad Formativa (IDF)

No todos los artículos generan la misma cantidad de aprendizajes.

Propongo calcular un **Índice de Densidad Formativa (IDF)** para estimar la carga curricular derivada de cada disposición.

### Variables

* Número de ENF identificados.
* Nivel de criticidad.
* Complejidad técnica.
* Frecuencia de aplicación.
* Riesgo asociado.

### Clasificación

| IDF | Interpretación                                              |
| --- | ----------------------------------------------------------- |
| 1   | Baja densidad (orientación general).                        |
| 2   | Densidad moderada (uno o dos resultados de aprendizaje).    |
| 3   | Alta densidad (múltiples competencias relacionadas).        |
| 4   | Muy alta densidad (requiere unidad didáctica propia).       |
| 5   | Crítica (requiere módulo completo y evaluación específica). |

Este índice servirá para justificar la duración de los cursos y la profundidad con la que se abordarán determinados temas.

# 11.7 Relación con el EC0060

Una de las fortalezas del proyecto será que cada ENF se confrontará con los elementos del **EC0060**:

* Desempeños esperados.
* Productos.
* Conocimientos.
* Actitudes.
* Hábitos de trabajo.
* Valores.

Cuando exista correspondencia, se documentará expresamente en la ficha. Si se identifica un requisito legal que no esté cubierto por el estándar, se señalará como una **brecha competencial**, para incorporarlo al diseño curricular.

---

# 11.8 Control de Cobertura

Cada ENF recibirá un estado dentro del sistema:

| Estado               | Significado                                    |
| -------------------- | ---------------------------------------------- |
| Identificado         | Se detectó el requisito normativo.             |
| Analizado            | Se elaboró la ficha FENF.                      |
| Competencia definida | Existe competencia asociada.                   |
| Curso asignado       | Se integró a un programa de capacitación.      |
| Evaluación diseñada  | Cuenta con instrumentos de evaluación.         |
| Implementado         | Forma parte de la capacitación institucional.  |
| Verificado           | Existen evidencias documentales de aplicación. |

Este control permitirá conocer, en cualquier momento, el grado de avance del proyecto y la cobertura real de las obligaciones formativas.

# Propuesta estratégica para la siguiente etapa

Considero que hemos llegado al punto en el que la metodología está suficientemente consolidada. A partir del próximo desarrollo iniciaremos el **análisis exhaustivo de las disposiciones normativas**, comenzando por aquellas que tienen mayor impacto sobre la actuación del personal operativo y sobre la obligación de capacitar, independientemente del documento en que se encuentren. En lugar de seguir el orden secuencial de cada ley, agruparemos los requisitos por grandes ejes funcionales —marco jurídico, principios de actuación, control de accesos, vigilancia, comunicación, gestión documental, atención a usuarios, emergencias, supervisión y actualización profesional—. Esta reorganización permitirá construir un sistema curricular coherente, evitar duplicidades entre las distintas normas y mantener una trazabilidad completa hacia cada artículo y cada apartado del EC0060.

Éste será el inicio de la producción del contenido técnico definitivo que alimentará simultáneamente los seis tomos de la colección. A partir de aquí, cada entregable incrementará directamente el patrimonio documental del proyecto y reducirá la distancia hasta disponer de un sistema integral, implementable y jurídicamente defendible para la profesionalización del personal operativo de empresas de seguridad privada.
