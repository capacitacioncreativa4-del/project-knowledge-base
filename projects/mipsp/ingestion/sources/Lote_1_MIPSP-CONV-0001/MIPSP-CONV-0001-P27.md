Se preguntan:

> **¿Qué debe aprender un guardia?**

Ese pequeño cambio cambia completamente la ingeniería curricular.

# TOMO II

# CAPÍTULO 6

# Inventario Analítico de Requisitos Formativos (IARF)

## PARTE I

## Metodología de extracción normativa

Antes de iniciar el análisis artículo por artículo debemos establecer una regla de ingeniería jurídica.

No toda disposición legal genera capacitación.

Por ejemplo:

Una norma puede referirse a:

* facultades de la autoridad;
* procedimientos administrativos;
* requisitos para obtener autorización;
* sanciones;
* obligaciones del representante legal.

Todo ello es importante jurídicamente.

Pero **no necesariamente genera competencias para el personal operativo**.

Por ello cada disposición será clasificada.

---

# Nivel A

## Requisito Formativo Directo (RFD)

Genera capacitación obligatoria.

Ejemplos:

* actuación del personal;
* uso de equipo;
* control de accesos;
* rondines;
* radiocomunicación;
* bitácoras;
* atención al usuario.

Estos artículos pasarán íntegramente al sistema curricular.

---

# Nivel B

## Requisito Formativo Indirecto (RFI)

No obliga directamente al guardia, pero condiciona su actuación.

Ejemplos:

* obligaciones del prestador;
* supervisión;
* manuales;
* registros.

Estos artículos producirán competencias para supervisores, instructores o responsables operativos.

---

# Nivel C

## Requisito Administrativo (RA)

Genera procedimientos administrativos, pero no contenidos curriculares para el personal operativo.

Ejemplos:

* autorizaciones;
* renovación de permisos;
* avisos administrativos.

No integrarán programas para guardias, aunque sí podrán incorporarse a cursos para personal directivo o administrativo.

---

# Nivel D

## Requisito Estratégico (RE)

No produce cursos específicos, pero determina políticas institucionales.

Ejemplos:

* principios;
* coordinación;
* obligaciones generales del prestador.

Estos requisitos alimentarán el Sistema de Gestión desarrollado en el Tomo VI.

---

# ¿Qué logramos con esta clasificación?

Evitamos un problema frecuente en muchos programas de capacitación: incluir contenidos que no guardan relación con el puesto del elemento operativo.

Cada hora de formación deberá estar justificada por un requisito identificado como RFD o, cuando corresponda, por un RFI claramente asociado a funciones de supervisión o gestión.

# CAPÍTULO 7

# Matriz de Extracción de Competencias (MEC)

Cada artículo de las cinco fuentes será procesado mediante una matriz normalizada.

## Ejemplo de estructura

| Campo                  | Contenido                                    |
| ---------------------- | -------------------------------------------- |
| Código FARF            | FARF-0001                                    |
| Fuente                 | Ley / Reglamento / EC0060                    |

| Tipo de requisito      | RFD / RFI / RA / RE                          |
| Tema                   | Síntesis del contenido                       |
| Competencias derivadas | Codificadas                                  |
| Procesos asociados     | CMPO                                         |
| Riesgos mitigados      | Clasificados                                 |

| Evidencia              | Conocimiento, desempeño o producto           |
| Instrumento            | Examen, rúbrica, lista de cotejo, simulación |
| Indicador              | Código SIEC                                  |

# CAPÍTULO 9

# Catálogo Maestro de Evidencias (CME)

Las normas y el EC0060 no sólo exigen conocimientos; también requieren demostrar el desempeño. Por ello, el sistema clasificará las evidencias en cuatro categorías:

### E1. Evidencias de conocimiento

* Exámenes escritos.
* Casos prácticos.
* Análisis normativo.

### E2. Evidencias de desempeño

* Observación directa.
* Simulaciones.
* Prácticas supervisadas.

### E3. Evidencias documentales

* Bitácoras.
* Informes.
* Registros.
* Reportes.

### E4. Evidencias de resultado

* Indicadores de desempeño.
* Auditorías.
* Retroalimentación de clientes.
* Disminución de incidentes atribuibles a errores de actuación.

Esta última categoría constituye una aportación metodológica relevante: permite evaluar no sólo el aprendizaje inmediato, sino el impacto real de la capacitación sobre la operación.

# CAPÍTULO 10

# Índice de Cobertura Curricular (ICC-N)

Propongo incorporar un indicador global que mida el grado de cumplimiento del sistema respecto de las obligaciones identificadas.

La fórmula sería:

[
ICC\text{-}N = \frac{\text{Requisitos Formativos Cubiertos}}{\text{Requisitos Formativos Identificados}} \times 100
]

Donde:

* **Requisitos Formativos Identificados (RFI):** total de obligaciones de capacitación extraídas de las cinco fuentes.
* **Requisitos Formativos Cubiertos (RFC):** aquellas que cuentan con competencia, curso, evaluación y evidencia.

La meta institucional debería ser:

* **100 % de cobertura normativa** para los requisitos clasificados como RFD.
* **Cobertura documentada** para los RFI aplicables.
* Justificación formal para los RA y RE que no generen capacitación operativa.

Este indicador permitirá demostrar, con un criterio objetivo, que el sistema de capacitación cubre íntegramente las obligaciones legales.

# Propuesta estratégica para la siguiente fase

Considero que hemos concluido la arquitectura metodológica necesaria. El siguiente paso ya no consiste en diseñar nuevos modelos, sino en **producir el contenido técnico definitivo**.

Mi propuesta es iniciar el **Inventario Analítico de Requisitos Formativos** mediante el análisis secuencial y exhaustivo de las cinco fuentes normativas. Trabajaremos disposición por disposición, elaborando una FARF para cada requisito identificado y alimentando simultáneamente:

* la Matriz de Cobertura Normativa;
* el Diccionario Maestro de Competencias;
* el Catálogo Maestro de Procesos Operativos;
* el Catálogo Maestro de Programas de Capacitación;
* y el Sistema Integral de Evaluación.

Con este enfoque, al finalizar el proyecto podremos afirmar, con respaldo documental y trazabilidad completa, que **cada obligación legal relacionada con la formación del personal operativo ha sido identificada, interpretada, transformada en competencia, incorporada a un programa de capacitación y asociada a evidencias verificables de cumplimiento**. Ese será el fundamento técnico que dará solidez jurídica, pedagógica y operativa a toda la colección.
