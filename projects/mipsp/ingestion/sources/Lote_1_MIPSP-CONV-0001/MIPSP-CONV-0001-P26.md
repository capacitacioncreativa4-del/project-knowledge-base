# TOMO II

# CAPÍTULO 5

# Inventario Analítico de Requisitos Formativos (IARF)

## 5.1 Definición

El IARF es la identificación, clasificación y descomposición sistemática de **todos los requisitos de formación, capacitación, actualización, evaluación y profesionalización** que derivan de las cinco fuentes documentales analizadas.

Su finalidad es garantizar que exista una relación uno a uno entre:

**Norma → Competencia → Curso → Evaluación → Evidencia → Indicador.**

---

# 5.2 Estructura del IARF

Cada requisito se documentará mediante una **Ficha Analítica de Requisito Formativo (FARF)**.

## Encabezado

* Código único.
* Fuente normativa.
* Artículo.
* Fracción.
* Inciso.
* Página del documento fuente (para efectos de trazabilidad documental).
* Tipo de disposición (obligación, prohibición, facultad, principio, requisito, procedimiento).

---

## Análisis jurídico

No se limitará a reproducir el texto legal.

Se documentará:

* finalidad de la disposición;
* sujetos obligados;
* alcance;
* condiciones de aplicación;
* consecuencias del incumplimiento.

# 5.4 Matriz de Trazabilidad Normativa (MTN)

Aquí aparece uno de los componentes que, desde mi experiencia, será el más útil durante auditorías, verificaciones y procesos de autorización.

Cada requisito formativo tendrá una trazabilidad completa.

| Campo                    | Ejemplo      |
| ------------------------ | ------------ |
| Código FARF              | FARF-0001    |
| Norma                    | Ley Federal  |
| Artículo                 | X            |
| Competencia              | COMP-JUR-001 |
| Curso                    | CUR-001      |
| Unidad                   | U1           |
| Resultado de aprendizaje | RA-001       |
| Instrumento              | LC-001       |
| Evidencia                | ED-001       |
| Indicador                | IND-001      |

Con esta matriz será posible responder inmediatamente preguntas como:

* ¿Qué curso atiende el artículo "X"?
* ¿Qué evidencia demuestra su cumplimiento?
* ¿Qué instrumento la evalúa?
* ¿Qué indicador permite verificar su eficacia?

# Propuesta de un componente adicional: Ontología de Competencias

Quisiera incorporar un elemento que rara vez se desarrolla en sistemas de capacitación tradicionales y que facilitará enormemente el mantenimiento futuro del proyecto.

Propongo construir una **Ontología de Competencias para la Seguridad Privada (OCSP)**.

No se tratará únicamente de un catálogo, sino de un modelo de relaciones entre conceptos.

Por ejemplo:

```text
Legalidad
│
├── Facultades
│      ├── Control de accesos
│      ├── Verificación de identidad
│      └── Registro de personas
│
├── Límites de actuación
│      ├── Derechos humanos
│      ├── Uso proporcional de medios
│      └── Protección de datos
│
└── Responsabilidades
       ├── Administrativas
       ├── Civiles
       └── Penales
```

Esta estructura permitirá:

* evitar duplicidades entre cursos;
* identificar competencias transversales;
* analizar el impacto de una reforma legal sobre varios programas simultáneamente;
* incorporar nuevas modalidades de servicio sin rediseñar toda la arquitectura.

## La siguiente fase del proyecto

A partir del siguiente entregable propongo iniciar el **desarrollo exhaustivo del IARF**, utilizando exclusivamente los cinco documentos fuente. Analizaremos cada disposición normativa aplicable y construiremos sus correspondientes FARF, con referencias precisas a los artículos, fracciones e incisos. Paralelamente alimentaremos la Matriz de Trazabilidad Normativa, el Diccionario Maestro de Competencias y el Catálogo Maestro de Procesos Operativos.

En mi criterio, ése será el núcleo técnico de toda la obra. Cuando concluyamos esa fase, el resto de los tomos (programas de capacitación, manuales, evaluaciones, indicadores y procedimientos) podrá generarse de forma sistemática, con la certeza de que **ningún requisito legal o competencial habrá quedado sin cobertura**. Ese nivel de exhaustividad es el que permitirá que el sistema sea sólido tanto desde la perspectiva pedagógica como desde la jurídica y la operativa.
