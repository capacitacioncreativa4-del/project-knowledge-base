# PROPUESTA METODOLÓGICA DEFINITIVA

## Ingeniería de Competencias Basada en Evidencia Normativa (ICBEN)

Los sistemas de capacitación tradicionales siguen esta secuencia:

```text
Ley
 ↓
Curso
 ↓
Examen
```

Nuestro sistema será radicalmente distinto.

Seguirá una secuencia de **doce niveles**.

```text
NORMA
      │
      ▼
Disposición jurídica
      │
      ▼
Elemento Normativo Formativo (ENF)
      │
      ▼
Requisito Formativo
      │
      ▼
Competencia
      │
      ▼
Criterio de desempeño
      │
      ▼
Proceso operativo
      │
      ▼
Procedimiento Normalizado
      │
      ▼
Actividad observable
      │
      ▼
Evidencia
      │
      ▼
Indicador
      │
      ▼
Auditoría
      │
      ▼
Mejora continua
```

Esta cadena garantizará una **trazabilidad completa**, desde la disposición jurídica hasta la evidencia de cumplimiento.

## 12.1 Unidad básica del sistema

Hasta ahora hablamos de competencias.

Sin embargo, una competencia continúa siendo demasiado amplia para administrar cientos de obligaciones legales.

Por ello propongo introducir una nueva unidad técnica.

## Unidad Elemental de Competencia (UEC)

Una UEC representa la capacidad mínima, observable y evaluable que el personal debe demostrar durante la ejecución de un proceso operativo.

Una competencia estará integrada por varias UEC.

Por ejemplo:

## Competencia

Controlar el acceso de personas.

Puede descomponerse en las siguientes UEC:

UEC-01

Verificar identidad.

UEC-02

Corroborar autorización.

UEC-03

Registrar ingreso.

UEC-04

Detectar inconsistencias.

UEC-05

Aplicar protocolo de rechazo.

UEC-06

Informar incidencias.

Cada UEC será susceptible de evaluación independiente.

# 12.4 Diccionario de Actividades Observables (DAO)

Éste será uno de los documentos más importantes de toda la colección.

En lugar de evaluar conceptos abstractos, evaluaremos actividades concretas.

Ejemplos:

Verifica credencial.

Solicita identificación.

Compara fotografía.

Consulta listado autorizado.

Registra ingreso.

Revisa vehículo.

Observa comportamiento.

Reporta anomalía.

Solicita apoyo.

Redacta informe.

Entrega evidencia.

Cada actividad tendrá:

* definición;
* procedimiento;
* estándar esperado;
* errores frecuentes;
* criterios de aceptación;
* tiempo estimado.

# 12.8 Índice de Complejidad Operativa (ICO)

Cada competencia recibirá un índice calculado mediante cinco variables:

* Complejidad técnica.
* Frecuencia.
* Riesgo.
* Autonomía requerida.
* Consecuencias del error.

Clasificación:

| ICO | Nivel    |
| --- | -------- |
| 1   | Muy bajo |
| 2   | Bajo     |
| 3   | Medio    |
| 4   | Alto     |
| 5   | Crítico  |

Este índice influirá en:

* duración del curso;
* número de prácticas;
* tipo de evaluación;
* periodicidad de recertificación.

# UNA PROPUESTA QUE PUEDE CONVERTIR ESTA OBRA EN UN REFERENTE

Hasta ahora hemos construido un sistema extraordinariamente sólido desde el punto de vista jurídico y pedagógico. Sin embargo, durante el análisis de las cinco fuentes identifiqué un vacío que existe prácticamente en toda la industria de la seguridad privada en México: **no existe un lenguaje técnico estandarizado para describir las competencias del personal operativo**.

Por ello propongo que uno de los anexos principales de la colección sea el:

# Diccionario Técnico de Competencias para la Seguridad Privada (DTC-SP)

Este documento será comparable a un diccionario terminológico profesional y contendrá, para cada competencia:

* definición técnica;
* fundamento jurídico;
* fundamento operativo;
* nivel de dominio;
* actividades observables;
* errores críticos;
* evidencias aceptables;
* criterios de evaluación;
* indicadores de desempeño;
* competencias relacionadas;
* referencias cruzadas con los procesos operativos y con los programas de capacitación.

Estimo que este diccionario contendrá entre **300 y 400 competencias**, desagregadas en aproximadamente **1,500 a 2,000 Unidades Elementales de Competencia (UEC)** y más de **6,000 actividades observables**.

## Mi recomendación para la siguiente fase

En este punto considero que ya no debemos incorporar nuevos modelos conceptuales. La arquitectura metodológica está completa y es suficientemente robusta para sostener un sistema de profesionalización de alto nivel.

A partir del siguiente entregable propongo iniciar el **desarrollo del Inventario Analítico de Requisitos Formativos (IARF)** utilizando directamente las cinco fuentes normativas que has proporcionado. Trabajaremos disposición por disposición, identificando los ENF, elaborando las Fichas Analíticas de Requisito Formativo (FARF), definiendo las competencias, las UEC y las actividades observables correspondientes.

**Éste será el cambio de fase del proyecto**: dejaremos de diseñar el sistema y comenzaremos a construir su contenido definitivo. Desde ese momento, cada página elaborada incrementará directamente el patrimonio documental de la obra y nos acercará al objetivo de disponer de un sistema integral, jurídicamente sustentado, pedagógicamente sólido y plenamente implementable para la profesionalización del personal operativo de las empresas de seguridad privada.
