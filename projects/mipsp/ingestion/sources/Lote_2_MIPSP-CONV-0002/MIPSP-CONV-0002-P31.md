# 1. Propósito de la sesión

Desarrollar la competencia para transmitir información operativa de manera clara, precisa, oportuna y verificable, utilizando protocolos institucionales de comunicación oral, escrita y por radiocomunicación, así como elaborar reportes y registros documentales que garanticen la trazabilidad de las operaciones.

---

# 3. Resultados de aprendizaje

Al finalizar la sesión, el participante será capaz de:

* comunicar información operativa siguiendo el Protocolo Institucional de Comunicación Operativa (PICO);
* elaborar reportes claros, cronológicos y objetivos;
* registrar hechos relevantes en bitácoras institucionales;
* emplear correctamente la radiocomunicación;
* evitar ambigüedades, interpretaciones subjetivas y omisiones.

---

# 4. Principios de la comunicación operativa

Toda comunicación institucional deberá cumplir con los siguientes principios:

1. **Exactitud:** transmitir únicamente información confirmada.
2. **Claridad:** utilizar lenguaje sencillo, técnico y sin ambigüedades.
3. **Oportunidad:** comunicar en el momento adecuado.
4. **Brevedad:** emplear únicamente la información necesaria.
5. **Trazabilidad:** dejar evidencia documental cuando corresponda.
6. **Confidencialidad:** proteger la información sensible conforme a la política institucional.

---

# 5. Protocolo Institucional de Comunicación Operativa (PICO)

El PICO establece una secuencia uniforme para toda comunicación relevante:

| Etapa          | Acción                                     |
| -------------- | ------------------------------------------ |
| Identificar    | Identificarse y verificar al receptor.     |
| Contextualizar | Precisar ubicación, fecha y hora.          |
| Informar       | Describir objetivamente los hechos.        |
| Confirmar      | Verificar que el mensaje fue comprendido.  |
| Registrar      | Documentar la comunicación cuando proceda. |

# 8. Guion Operativo del Instructor (GOI)

## Actividad inicial

El instructor leerá dos reportes sobre un mismo hecho:

**Reporte A**

> "Un individuo sospechoso estuvo rondando el acceso y parecía querer entrar."

**Reporte B**

> "A las 09:15 horas se observó a un masculino adulto permanecer durante aproximadamente cinco minutos frente al acceso principal. Solicitó información sobre el horario de atención y se retiró sin intentar ingresar."

El grupo analizará cuál reporte cumple con los principios institucionales y por qué.

---

# 9. Desarrollo conceptual

Se explicarán las diferencias entre:

| Incorrecto           | Correcto                                                               |
| -------------------- | ---------------------------------------------------------------------- |
| Opinión              | Hecho verificable                                                      |
| "Parecía nervioso"   | "Presentaba temblor visible en las manos y evitaba el contacto visual" |
| "Fue grosero"        | "Elevó el tono de voz y utilizó expresiones ofensivas"                 |
| "Todo estaba normal" | "No se detectaron anomalías durante el periodo observado"              |

---

# 10. Radiocomunicación

## Reglas institucionales

* Identificarse antes de transmitir.
* Esperar canal libre.
* Evitar conversaciones innecesarias.
* Utilizar lenguaje técnico.
* Confirmar recepción.
* Registrar comunicaciones críticas.

### Ejemplo

> **Puesto Alfa a Supervisor Uno. Solicito confirmación de ingreso del proveedor identificado como Transportes del Centro, unidad TC-214. Cambio.**

> **Supervisor Uno a Puesto Alfa. Confirmado. Autorizado el ingreso conforme a la orden de servicio 4587. Cambio y fuera.**

---

# 11. Elaboración de reportes

Todo reporte deberá responder, como mínimo, a las siguientes preguntas:

* ¿Qué ocurrió?
* ¿Cuándo ocurrió?
* ¿Dónde ocurrió?
* ¿Quién participó?
* ¿Cómo ocurrió?
* ¿Qué acciones se realizaron?
* ¿Cuál fue el resultado?
* ¿Qué seguimiento se requiere?

---

# 12. Taller práctico

Cada participante elaborará tres documentos:

1. Registro de bitácora.
2. Reporte operativo.
3. Informe breve de incidencia.

Posteriormente realizará una comunicación por radio basada en el mismo escenario.

# 14. Matriz de Riesgos Operativos (MRO)

| -------------------------------- | ------------ | ------- | ---------------------------------- |

# 15. Matriz de Errores Críticos (MEC)

| Error                                 | Consecuencia            | Acción correctiva                   |
| ------------------------------------- | ----------------------- | ----------------------------------- |
| Utilizar opiniones como hechos        | Decisiones erróneas     | Reescritura del reporte             |
| Omitir fecha u hora                   | Pérdida de trazabilidad | Corrección inmediata                |
| No confirmar recepción del mensaje    | Fallas de coordinación  | Repetición del ejercicio            |
| No registrar una comunicación crítica | Debilidad en auditoría  | Integración inmediata a la bitácora |
