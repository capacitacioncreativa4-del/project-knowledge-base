# ETS-CUR001-MOD03-SES01

## Expediente Técnico de Sesión

**Duración:** 4 horas

**Versión:** 1.0

**Clasificación:** Documento Operativo Institucional

---

# 1. Propósito de la sesión

Desarrollar la capacidad del participante para observar de manera sistemática el entorno operativo, identificar condiciones y comportamientos que puedan representar riesgos para la seguridad, evaluar preliminarmente su impacto y comunicar oportunamente las anomalías conforme a los procedimientos institucionales.

---

# 2. Competencias asociadas

| Código      | Competencia                                       |
| ----------- | ------------------------------------------------- |
| COM-OPS-006 | Detectar condiciones y comportamientos de riesgo. |
| COM-OPS-007 | Comunicar oportunamente situaciones relevantes.   |
| COM-OPS-010 | Aplicar técnicas de observación sistemática.      |
| COM-OPS-011 | Registrar anomalías con objetividad y precisión.  |

---

# 3. Resultados de aprendizaje

Al concluir la sesión el participante será capaz de:

* aplicar una metodología estructurada de observación;
* distinguir entre hechos, interpretaciones y suposiciones;
* identificar indicadores tempranos de riesgo;
* clasificar anomalías conforme a su prioridad;
* elaborar reportes objetivos y verificables;
* comunicar oportunamente una condición de riesgo.

---

# 4. Fundamento operativo

Uno de los principios rectores del MIPSP establece:

> **"No toda anomalía constituye un incidente, pero prácticamente todo incidente estuvo precedido por una anomalía observable."**

En consecuencia, el personal operativo debe desarrollar competencias de observación profesional antes que capacidades meramente reactivas.

# 6. Modelo Institucional de Observación Operativa (MIOO)

El MIPSP adopta un modelo de observación en cinco fases:

| Fase        | Objetivo                                |
| ----------- | --------------------------------------- |
| Observar    | Percibir sistemáticamente el entorno.   |
| Identificar | Reconocer condiciones inusuales.        |
| Verificar   | Confirmar objetivamente la información. |
| Valorar     | Estimar el nivel de riesgo.             |
| Comunicar   | Informar conforme al procedimiento.     |

Este modelo se aplicará transversalmente en todos los cursos operativos.

# 8. Guion Operativo del Instructor (GOI)

## Actividad inicial

El instructor proyectará durante 30 segundos una imagen de un acceso con múltiples elementos.

Posteriormente retirará la imagen y formulará preguntas como:

* ¿Cuántas personas observó?
* ¿Había algún objeto fuera de lugar?
* ¿Qué llamó más su atención?
* ¿Qué dato considera relevante para la seguridad?

El objetivo será demostrar que observar no equivale a mirar.

---

# 9. Desarrollo conceptual

Se explicarán las diferencias entre:

| Concepto    | Definición                             |
| ----------- | -------------------------------------- |
| Hecho       | Información objetivamente verificable. |
| Observación | Registro estructurado de un hecho.     |
| Inferencia  | Interpretación basada en indicios.     |
| Suposición  | Conclusión sin evidencia suficiente.   |

Se insistirá en que los reportes institucionales deben contener hechos verificables y distinguir claramente cualquier interpretación.

---

# 10. Indicadores de riesgo

Los participantes aprenderán a identificar, entre otros:

### Personas

* Conducta evasiva.
* Nerviosismo inusual.
* Intentos reiterados de ingreso.
* Información contradictoria.
* Permanencia injustificada en zonas de acceso.

### Vehículos

* Placas alteradas o inconsistentes.
* Carga visible no documentada.
* Permanencia prolongada en áreas restringidas.
* Maniobras inusuales.

### Instalaciones

* Cerraduras dañadas.
* Iluminación deficiente.
* Cámaras fuera de servicio.
* Señalización alterada.
* Barreras abiertas sin autorización.

---

# 11. Taller práctico

Se organizarán cuatro estaciones de observación.

Cada estación contendrá entre cinco y ocho anomalías deliberadamente incorporadas.

El participante dispondrá de cinco minutos para:

1. Observar.
2. Registrar.
3. Clasificar.
4. Reportar.

```text
 │     ▼

 └── Sí

¿Puede verificarse objetivamente?
       │
 ├── No

 │
 └── Sí
       ▼
Clasificar nivel de riesgo
       │
 ├── Bajo → Registrar
 ├── Medio → Informar al supervisor
 └── Alto → Activar protocolo correspondiente
```

# 13. Matriz de Riesgos Operativos (MRO)

| Riesgo                               | Probabilidad | Impacto | Control                   |
| ------------------------------------ | ------------ | ------- | ------------------------- |
| No detectar una anomalía             | Media        | Alto    | Observación sistemática   |
| Reportar información incompleta      | Media        | Medio   | Uso del FRA-001           |
| Clasificar incorrectamente un riesgo | Baja         | Alto    | Validación por supervisor |
| Confundir hechos con opiniones       | Media        | Medio   | Capacitación continua     |

---

# 14. Matriz de Errores Críticos (MEC)

| Error                                   | Consecuencia            | Corrección                               |
| --------------------------------------- | ----------------------- | ---------------------------------------- |
| Reportar suposiciones como hechos       | Pérdida de credibilidad | Revisión del reporte y retroalimentación |
| No comunicar una anomalía relevante     | Incremento del riesgo   | Simulación correctiva                    |
| Omitir datos objetivos                  | Deficiencia documental  | Repetición del ejercicio                 |
| Clasificar incorrectamente la prioridad | Respuesta inadecuada    | Análisis de casos                        |
