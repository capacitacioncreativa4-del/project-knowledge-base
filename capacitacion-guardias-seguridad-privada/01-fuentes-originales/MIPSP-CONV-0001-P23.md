# TOMO IV

# ARQUITECTURA CURRICULAR Y SISTEMA INTEGRAL DE APRENDIZAJE

## Capítulo 1

# Modelo Curricular Basado en Riesgos, Competencias y Desempeño (MCRCD)

## 1.1 Principio rector

El sistema curricular no se organizará por asignaturas, sino por la combinación de tres variables:

* Obligaciones normativas.
* Procesos operativos.
* Riesgos asociados al desempeño.

La consecuencia es que cada hora de capacitación tendrá una justificación objetiva.

No habrá contenidos incorporados por tradición o costumbre.

Todo contenido deberá responder, al menos, a una de estas preguntas:

* ¿Qué obligación legal satisface?
* ¿Qué proceso operativo fortalece?
* ¿Qué riesgo ayuda a prevenir?
* ¿Qué evidencia permitirá demostrar la competencia?

## Sección II. Justificación

No se limitará a una introducción general.

Documentará expresamente:

* fundamento jurídico;
* fundamento operativo;
* fundamento técnico;
* fundamento pedagógico;
* riesgos mitigados;
* competencias desarrolladas.

## Sección III. Perfil de ingreso

## Sección IV. Perfil de egreso

# Propuesta innovadora: Índice de Criticidad Curricular (ICC)

Quiero proponerte un componente que no suele encontrarse en los sistemas tradicionales y que aporta una base objetiva para decidir la intensidad de la capacitación.

Cada unidad didáctica recibirá un **Índice de Criticidad Curricular (ICC)** calculado a partir de cinco factores:

| Factor                  | Peso sugerido |
| ----------------------- | ------------: |
| Riesgo para personas    |          30 % |
| Riesgo jurídico         |          25 % |
| Riesgo patrimonial      |          20 % |
| Frecuencia de ejecución |          15 % |
| Complejidad técnica     |          10 % |

Con base en el puntaje obtenido, las unidades se clasificarán así:

| Nivel ICC | Tratamiento curricular                                  |
| --------- | ------------------------------------------------------- |
| ICC-1     | Exposición teórica suficiente.                          |
| ICC-2     | Teoría y ejercicios prácticos.                          |
| ICC-3     | Prácticas supervisadas obligatorias.                    |
| ICC-4     | Simulación con evaluación formal.                       |
| ICC-5     | Certificación y recertificación periódica obligatorias. |

Este modelo permitirá justificar, por ejemplo, por qué el control de accesos o la gestión de incidentes requieren más horas de práctica que otros temas de menor impacto.

# Hacia una Academia Corporativa

Considero que el proyecto puede ir un paso más allá. En lugar de limitarse a un catálogo de cursos, propongo que los seis tomos sirvan de base para crear una **Academia Corporativa de Seguridad Privada**. Esta academia no sería un edificio, sino un modelo de gestión del conocimiento con procesos definidos para el diseño curricular, la impartición, la evaluación, la certificación, la actualización normativa, la gestión de instructores, el control documental y la mejora continua.

El resultado sería un sistema capaz de adaptarse a futuras reformas legales, nuevas modalidades de servicio y cambios tecnológicos sin necesidad de rediseñar toda la oferta formativa.

## El siguiente hito del proyecto

A partir del próximo entregable propongo iniciar el **Tomo V: Sistema Integral de Evaluación y Certificación por Competencias**. En él construiremos un modelo que no sólo evalúe si el participante aprobó un curso, sino si **demuestra de manera consistente el desempeño esperado en condiciones reales de operación**. Desarrollaremos un marco de evaluación multinivel, bancos de reactivos vinculados con la normativa, rúbricas de desempeño, listas de cotejo para observación en campo, simulaciones calificadas y un sistema de indicadores que permita medir el impacto de la capacitación sobre la calidad del servicio y el cumplimiento normativo. En mi opinión, ese tomo convertirá la colección en un verdadero sistema de aseguramiento de la competencia profesional.
